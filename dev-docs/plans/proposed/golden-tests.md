## Summary

Implement a YAML-driven golden test runner for clank.sh. Tests are declared as plain
YAML files. A build script generates one `#[test]` per file at compile time. A
`clank-golden` helper crate contains all shared runner logic. No Rust is required to
add a new golden test.

## Developer Feedback

- Tests are partial by default (no `exit_code`): feed input, assert stdout/stderr, terminate.
- Tests with `exit_code` present are complete: feed input, send EOF, wait for exit, assert.
- Setup chaining: a test may reference another YAML file via `setup`; that file's script
  runs first in the same shell instance. Chains are supported (setup of a setup).
- A separate `clank-golden` crate captures shared helpers.
- Use `include_str!` + build script to generate one `#[test]` per YAML file — avoids
  `harness = false` and minimises dependencies.
- Avoid unnecessary dependencies, but `serde_yaml` for YAML parsing is acceptable.

## YAML Schema

```yaml
description: human-readable label (becomes the test name in cargo test output)
setup: relative/path/to/setup-file.yaml   # optional
script: |
  one or more lines of shell input
stdout: |                                  # optional — omit to skip stdout assertion
  expected exact stdout
stderr: |                                  # optional — omit to skip stderr assertion
  expected exact stderr
exit_code: 0                               # optional — presence makes this a complete test
```

**Partial test** (`exit_code` absent): runner feeds all input lines, asserts on
stdout/stderr captured so far, then terminates the shell instance.

**Complete test** (`exit_code` present): runner feeds all input lines, sends EOF,
waits for the shell to exit, then asserts exit code and stdout/stderr.

## `clank-golden` Crate

A library crate in the workspace. Contains:

- `GoldenTest` — struct deserialised from YAML via `serde_yaml`.
- `load(path: &str, content: &str) -> GoldenTest` — parse YAML from embedded string.
- `resolve_setup_chain(test: &GoldenTest, base_dir: &Path) -> Vec<String>` — recursively
  load setup scripts in order.
- `run(test: &GoldenTest, base_dir: &Path)` — build a shell, run setup chain, run script,
  capture output, assert, panic with diff on mismatch.

Dependencies for `clank-golden`: `serde`, `serde_yaml`, `clank` (lib).

## `build.rs` — Code Generation

At compile time, `build.rs` walks `clank/tests/golden/`, excludes `setup/`, and writes
`tests/golden_generated.rs` containing one `#[test]` per discovered YAML file:

```rust
// generated — do not edit
#[test]
fn golden_builtins_echo_hello() {
    clank_golden::run_embedded(
        include_str!("golden/builtins/echo-hello.yaml"),
        concat!(env!("CARGO_MANIFEST_DIR"), "/tests/golden/builtins"),
    );
}
```

The test name is derived from the file path (`/` and `-` replaced with `_`).

## Setup Chain

Setup files run in the **same `brush_core::Shell` instance** as the test script. This
is the only model that makes shell state (variables, working directory) available to
the test. Snapshotting is not supported.

A setup file may itself have a `setup` field, forming a chain. The resolver flattens
the chain depth-first before execution. Cycles are detected and cause a build-time or
test-time error.

Files under `golden/setup/` are excluded from test discovery by the build script —
they are never registered as test cases themselves.

## Acceptance Tests

1. `cargo test` runs all golden tests with zero failures alongside unit and integration tests.
2. Adding a new `.yaml` file to `clank/tests/golden/` and running `cargo test` discovers
   and runs it automatically with no Rust changes.
3. A test with `setup` correctly inherits shell state from the setup script.
4. A setup chain of depth 2 (A → B → test) resolves and executes correctly.
5. A partial test (no `exit_code`) asserts stdout/stderr and passes without waiting for exit.
6. A complete test (`exit_code` present) asserts exit code and stdout/stderr after EOF.
7. A stdout mismatch causes a test failure with a human-readable diff.
8. Omitting `stdout` skips the stdout assertion without failing.
9. Files under `golden/setup/` are not registered as test cases.
